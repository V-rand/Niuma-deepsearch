# 中国民事案件律师工作生命周期总览（面向 AgentOS）

## 1. 文档目的

本文档用于帮助 Agent/Codex 理解中国民事案件代理工作的长期运行方式。

这里描述的不是某一个具体案件，而是律师团队在一般民事案件中从接案到结案归档的完整生命周期。该生命周期不是固定流水线，而是一个长期持续、条件分支、期限驱动、文件驱动、事件驱动的工作系统。

本文档的目标不是教模型背诵法律程序，而是帮助模型理解：

- 一个案件是一个长期 session；
- 律师工作的核心不是单次回答，而是长期管理案件状态；
- 工作围绕事实、证据、文书、检索、沟通、期限和执行推进展开；
- 系统需要支持动态规划，而不是预设死板 workflow。

## 2. 全周期阶段

一个典型中国民事案件代理，通常会经历以下阶段：

1. 委托代理阶段
2. 案件分析阶段
3. 诉前保全阶段（可选）
4. 立案 / 应诉阶段
5. 庭前准备阶段
6. 庭审阶段
7. 庭后阶段
8. 履行 / 执行阶段
9. 上诉前准备阶段
10. 二审立案 / 答辩阶段
11. 二审庭审阶段
12. 二审庭后及执行阶段
13. 再审 / 抗诉阶段
14. 结案与归档阶段

并不是每个案件都会完整经过所有阶段，但系统必须支持这些可能性。

## 3. 律师工作的本质

律师工作并不是“接到问题 -> 输出答案”这种短任务模式，而是一种长期案件经营。

其核心活动包括：

- 了解和校正案件事实；
- 收集、整理、补强证据；
- 检索法律条文、案例、学术文章和其他外部材料；
- 形成、修订和更新诉讼策略；
- 起草和提交各类诉讼文书；
- 与客户、法院、对方持续沟通；
- 根据程序推进、调解结果、裁判结果和执行进展动态调整策略；
- 在结案时完成复盘、归档、原件归还和知识沉淀。

## 4. 案件不是线性流程，而是长期状态机

案件并非总是按固定顺序前进，可能随时发生以下分支：

- 诉前保全或不保全；
- 立案成功或不予受理；
- 被告答辩或不答辩；
- 调解成功或调解失败；
- 判决后自动履行或进入强制执行；
- 一审结束后上诉或不上诉；
- 二审结束后执行、再审或抗诉；
- 再审申请被驳回、再审启动、检察建议或抗诉。

因此，系统应将案件理解为“可演进的状态空间”，而不是固定动作序列。

## 5. 各阶段的核心意义

### 5.1 委托代理阶段
这一阶段的任务是接入案件并建立初始上下文。

核心活动包括：
- 与客户洽谈，了解基本案情；
- 评估收案风险；
- 进行利益冲突检索；
- 收集材料和补充材料；
- 输出谈话笔录、材料清单、诉讼策略分析报告；
- 办理委托代理手续，收取代理费；
- 确定律师团队并建立客户服务群。

这一阶段的目标不是立刻推进程序，而是完成案件启动和基础上下文建模。

### 5.2 案件分析阶段
这一阶段要把“客户叙述”转化为“可诉讼的事实-证据-法律结构”。

核心活动包括：
- 补充证据材料；
- 法律条文、案例、学术文章及其他数据检索；
- 团队内部分析；
- 形成诉讼方案；
- 向客户反馈策略，并根据反馈修订。

### 5.3 诉前保全阶段（可选）
如果担心对方转移财产，或需要紧急固定现状，律师可在起诉前申请财产/行为保全。

核心活动包括：
- 起草诉前保全申请书；
- 提供担保；
- 提供财产线索；
- 跟进法院裁定结果；
- 跟进保全措施实施；
- 必要时复议或解除保全；
- 若 30 日内未起诉，保全可能解除。

这一阶段强调紧急性、时效性、财产线索和法院沟通。

### 5.4 立案 / 应诉阶段
原告侧主要任务是起诉、交材料、交费、推进立案；被告侧主要任务是收到起诉状副本后及时答辩和举证。

这一阶段常见动作包括：
- 起草起诉状、答辩状；
- 提交证据材料和证据目录；
- 缴纳诉讼费；
- 提出保全、调查令、延期举证、回避、管辖权异议等程序申请；
- 向客户和内部团队汇报立案情况 / 答辩情况。

### 5.5 庭前准备阶段
这是把案件材料转换为庭审表达能力的阶段。

核心活动包括：
- 案情研讨；
- 类案检索；
- 制作庭审大纲；
- 模拟法庭（可选）；
- 庭审告知与协调；
- 统计客户是否参加庭审；
- 准备证据原件和交通预案；
- 评估和解意愿并准备和解方案。

### 5.6 庭审阶段
庭审是动态、高压力、实时反馈的阶段。

核心活动包括：
- 核查材料；
- 出庭；
- 核对证据原件；
- 质证和辩论；
- 回应法官问题；
- 保存庭审笔录和对方信息；
- 如有调解，审核和解协议；
- 庭后迅速形成庭审情况简报。

### 5.7 庭后阶段
庭后并非结束，而是重要的信息更新期。

核心活动包括：
- 提交补充证据；
- 提交代理意见 / 补充代理意见；
- 与法官沟通案件情况；
- 定期向客户同步进展；
- 收到裁判文书后提示上诉期限；
- 全案复盘。

### 5.8 履行 / 执行阶段
裁判生效后，案件可能进入履行或强制执行。

核心活动包括：
- 询问客户是否申请执行；
- 起草执行申请书；
- 执行立案；
- 联系执行法官；
- 财产线索普查；
- 推动冻结、扣划、拍卖等措施；
- 申请失信、限高、限制出境等威慑措施；
- 执行和解；
- 执行异议；
- 执行完毕或法院终结执行。

### 5.9 上诉前准备与二审阶段
如果一审结果不理想，需要重新分析一审裁判并形成二审策略。

核心活动包括：
- 调取一审卷宗；
- 解构一审裁判文书；
- 复盘双方诉辩主张；
- 识别一审问题；
- 制定二审诉讼策略；
- 起草上诉状或答辩状；
- 立案、缴费、跟进案件移送；
- 二审庭前准备、庭审和庭后处理。

### 5.10 再审 / 抗诉阶段
生效裁判存在严重问题时，可进入再审或抗诉路径。

核心活动包括：
- 在法定期限内提出再审申请；
- 向对方送达申请副本；
- 等待法院审查；
- 必要时寻求检察建议或抗诉。

### 5.11 结案与归档阶段
案件结束后仍需完成收尾和沉淀。

核心活动包括：
- 结案小结；
- 卷宗归档；
- 卷宗移送；
- 原件归还；
- 所内审批；
- 装卷；
- 形成复盘与知识资产。

## 6. 从 AgentOS 视角理解该工作系统

### 6.1 每个案件是一个长期 session
每个案件都有独立上下文、独立待办、独立事件、独立文件目录。

### 6.2 文件系统是核心而不是附件
律师工作依赖大量文书、证据、报告、记录、简报、申请书、清单、复盘报告。系统必须以文件为中心组织状态。

### 6.3 期限和提醒是硬约束
该工作高度依赖各种法定期限和管理期限，必须有 scheduler / interrupt 机制。

### 6.4 检索必须可追溯
检索结果不仅要摘要，还要保留原始内容和来源路径，避免信息失真。

### 6.5 主 Agent 负责动态规划
主 Agent 不应死板按流程执行，而应基于当前阶段、现有材料、期限风险、外部事件和客户反馈做动态规划。

## 7. 对 Codex/Agent 的理解要求

请将律师工作理解为：
- 长期案件 session 管理；
- 文书驱动、证据驱动、事件驱动的专业服务工作；
- 需要持续维护事实、策略、待办和期限；
- 不是一次性问答，也不是固定工作流图；
- 应通过工具和 skill 完成工作，而不是把业务流程写死在底层架构中。
