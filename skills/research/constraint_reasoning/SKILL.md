# Constraint Reasoning

Use this as a reasoning workbench when search is not moving an active constraint forward. Do not treat it as a report format; it is for deciding the next move.

## Core Move

For the active constraint, write 2-5 possible reasoning chains from already-known facts before searching again.

Each chain should say:
- known fact
- lens used
- inference
- supports / excludes / unclear
- whether search is still needed only for verification

## Lenses

| Constraint type | Lenses to try |
| --- | --- |
| associative | literal wording; translation / Chinese characters; etymology / surname origin; nationality / geography; biography / profession; cultural archetype; domain metaphor |
| linguistic | original wording; translation variants; characters; pronunciation / homophone; etymology; domain-specific meaning |
| geographic | birthplace / residence / registered origin; region hierarchy; landmarks; historical geography; cultural region |
| temporal | event date; approval / launch / production / publication distinction; calendar system; source publication date |
| relational | role membership; cast / team / organization relation; alias / translation; source-specific wording |
| causal | mechanism; necessary condition; sufficient condition; alternative explanation; counterexample |

## Stop Searching Signal

If two searches fail to resolve the same constraint and you have candidate facts available, switch to this skill before another web search. Search only after the chains identify a concrete missing fact.

## Common Errors

- Looking for an exact text match for an associative clue.
- Treating absence of search results as absence of a reasoning path.
- Keeping a candidate alive after a hard constraint fails.
- Using a source snippet as proof instead of verifying the specific constraint.
